# doc-sync: コード変更 → ドキュメント反映

## 目的

コードが変わったのにドキュメントが古いまま、というズレを解消する。
ユーザーが「ドキュメント更新して」と言ったら、このワークフローで動く。

---

## ワークフロー

### Step 1: 何が変わったか把握

```bash
git status --short
git diff HEAD
```

- 変更ファイルの一覧を確認
- diff で「何が追加・削除・変更されたか」を具体的に読む
- コミット済み変更が対象の場合は `git diff HEAD~1 HEAD` や `git show` を使う

### Step 2: 変更後のソースファイルを読む

diff だけでは不十分な場合、変更された主要ファイルを実際に読む。
特に以下は必ず確認:
- エントリポイント（`main.py` 等）の処理フロー
- 設定ファイル（`config.py` 等）の定数・ソース一覧
- 新規追加されたファイル（ファイル構成ドキュメントに要追記）
- モデル定義（フィールドの追加・削除）

### Step 3: ドキュメントを読む

更新対象のドキュメントを読む。このプロジェクトの対象:
- `README.md` — アーキテクチャ図、ファイル構成、セットアップ手順
- `spec.md` — 詳細仕様（ソース一覧、Geminiロジック、LINEメッセージ仕様、エラーハンドリング）

### Step 4: 差分を特定して更新

コードとドキュメントを照合し、**ズレている箇所だけ**を更新する。

#### よく更新が必要なセクション

| 変更の種類 | 更新箇所 |
|---|---|
| 新ファイル追加 | ファイル構成ツリー |
| ソース追加・削除 | アーキテクチャ図、ソース一覧表 |
| 処理フロー変更 | アーキテクチャ図のフロー、モジュール説明 |
| 設定定数追加 | 該当機能の仕様セクション |
| モデルフィールド変更 | データ構造セクション |
| 新機能追加 | 機能説明セクション |
| API/Webhook変更 | 該当サービスの仕様セクション |

#### 更新の原則

- **削除された機能はドキュメントからも削除**（コメントアウトではなく完全削除）
- **新規追加はドキュメントの対応セクションに追記**
- **変更された挙動は実装に合わせて書き直す**
- 関係ないセクションは触らない（スコープを広げすぎない）

## このプロジェクト固有の注意点

### アーキテクチャ図（README.md, spec.md 冒頭）

```
GitHub Actions (cron)
  ↓
Python スクリプト
  ├── <各フェッチャー>  ← main.py の gather() に渡されているものだけ列挙
  ├── フィルタ・重複排除
  ├── Cloudflare KV 読み込み
  ├── Gemini API で選定
  ├── LINE Messaging API で送信
  └── Cloudflare KV に保存
```

`main.py` の `asyncio.gather()` に渡されているフェッチャーが「実際に動くもの」。
config.py に定数だけある（import されていない）フェッチャーはアーキテクチャ図に含めない。

### ファイル構成

`src/fetchers/` 以下に `.py` ファイルが追加されたら必ず記載する。
逆に import から削除されたフェッチャーでも、ファイルが存在するなら構成には残す。

### 記事ソース一覧（spec.md）

`src/config.py` の `RSS_SOURCES` リストが正とする。
config.py に URLが定義されていても RSS_SOURCES に入っていなければ「RSS取得対象外」として記載しない。

### Gemini 選定ロジック

- `PREFERRED_TOPICS` が定義されていればシステムプロンプトと選定基準に反映
- フォールバックモデル（`GEMINI_FALLBACK_MODEL`）が定義されていれば仕様に記載
- `_prefilter_articles` が動いていればプリフィルタリングの説明を追加

## 出力の確認

更新後、変更したセクションを要約して報告する:

```
更新箇所:
- README.md: ファイル構成（devto_fetcher.py 追加）
- spec.md: ファイル構成、Geminiロジック（フォールバックモデル・優先トピック）、
           ユーザー嗜好システム（get_summary 変更）
```
