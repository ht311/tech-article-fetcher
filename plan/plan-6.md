# plan6: フォルダ構成改善案

作成日: 2026-04-19

## 背景
現状、プロジェクトのフォルダ構成が肥大化・散在し、関連ファイルの探索や保守性が低下している。目的は責務ごとの明確化と将来の拡張性を確保すること。

## 目標
- 関心ごとでフォルダを整理し可読性を向上
- 依存関係を簡潔にしテスト/デプロイを容易化
- 移行手順を最小リスクで実行

## 提案構成（例）
- src/               : アプリケーションコード（現状と同じ）
  - core/           : コアロジック
  - services/       : 外部連携・ビジネスロジック
  - web/            : API/ダッシュボード等のエントリ
  - cli/            : CLI 関連
- tests/             : 全テスト（ユニット/統合）
- docs/              : ドキュメント（README以外）
- infra/             : terraform / cloud config（cloudflare はここへ移動検討）
- tools/             : 開発ツール・スクリプト
- dashboard/         : （必要なら web と統合）

## マイグレーション手順（TODO）
1. 現状のフォルダマップを作成（差分確認用）
2. 一時ブランチで小さな移動を段階的に実行（1～2ディレクトリずつ）
3. 各移動ごとにテスト実行・CI確認
4. ドキュメントとインポート/起動手順を更新
5. レビュー後 main ブランチへマージ

## リスクと対策
- CI/インポートのパス破壊: 小刻みな移行とテストでカバー
- 外部参照: 移行前に参照検索・置換スクリプトを用意

## 次のアクション（短期）
- フォルダマップ作成
- 優先度の高い移動対象を決定
- 移行用 todo を作成して順次実施

(実行するときは各 TODO を issue に展開することを推奨します。)
